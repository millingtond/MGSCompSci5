‚ùì Why Object-Oriented Programming?

Problems with Procedural Programming

As programs grow larger, procedural programming faces several challenges:

Identify the Problem

Click on each scenario to reveal the procedural programming issue:

Scenario 1: Global Variables

A large program with 50+ procedures all accessing and modifying the same global variables.

Problem: Any procedure can change any variable, making debugging extremely difficult!
Scenario 2: Code Duplication

Similar code for handling students, teachers, and admin staff is written three times.

Problem: Violates DRY (Don't Repeat Yourself) principle - changes need to be made in multiple places!
Scenario 3: Tight Coupling

Changing one procedure requires modifying 15 other procedures that depend on it.

Problem: Poor modularity - changes cascade throughout the entire program!

How OOP Solves These Problems

Procedural Problems
  • Global data accessible everywhere
  • Functions separated from data
  • Difficult to model real-world
  • Code reuse is challenging
OOP Solutions
  • Data encapsulated in objects
  • Data and methods together
  • Natural real-world modeling
  • Inheritance enables reuse

üì¶ Object
An instance of a class. A specific entity with its own data values.
Example: playerJohn, enemyGoblin1
üìã Class
A blueprint or template for creating objects. Defines properties and methods.
Example: class Player, class Enemy
üìä Property/Attribute
Data stored within an object. The characteristics of an object.
Example: health, name, position
‚ö° Method
Functions that belong to a class. Actions an object can perform.
Example: attack(), move(), heal()

Quick Check: If "Car" is a class, what would "myCar" be?

Common Mistakes to Avoid

  • ‚ùå Confusing classes with objects (classes are templates, objects are instances)
  • ‚ùå Making everything public (we'll learn about encapsulation next lesson)
  • ‚ùå Creating classes for actions instead of entities (e.g., "Calculate" should be a method, not a class)
  • ‚ùå Forgetting that objects have both data AND behavior

üìù Summary & Final Quiz

Key Takeaways

  • Paradigm Shift: OOP focuses on objects rather than procedures
  • Real-World Modeling: Objects mirror real-world entities
  • Encapsulation: Data and methods are bundled together
  • Problem Solving: OOP addresses issues with procedural programming
  • Building Blocks: Classes (templates) create Objects (instances)

üìã Exam Practice Questions

Question 1 (4 marks)

Explain two advantages of using object-oriented programming compared to procedural programming when developing large software systems.

Mark Scheme:

Award 2 marks per advantage (1 for identifying, 1 for explanation):

  • Encapsulation - data and methods bundled together, protecting data integrity
  • Code reusability - through inheritance, reducing duplication
  • Modularity - objects are self-contained units, easier to maintain
  • Real-world modeling - intuitive representation of real entities

Question 2 (3 marks)

A programmer is creating a game with different types of vehicles. Identify three objects that might be needed and suggest one property for each.

Mark Scheme:

1 mark for each valid object with appropriate property. Examples:

  • Car - property: speed / fuel / color
  • Motorcycle - property: engineSize / brand
  • Truck - property: cargoCapacity / weight
  • Bicycle - property: gears / type
