💥 What are Exceptions?

Definition

An exception is a situation in code that causes a program to crash - it's a run-time error or fatal error. It is an unplanned event that we don't want to happen!

🎮 Exception Simulator

Let's see what happens when an exception occurs:

num2 = 0
result = num1 / num2
print("Result:", result)

When we run this code...

    result = num1 / num2

  • ❌ Stops execution immediately
  • ❌ Shows an unfriendly error message
  • ❌ Loses any unsaved data
  • ❌ Provides poor user experience

🛡️ Exception Handling

Exception Handling alerts the system to check for possible run-time errors and handles them without crashing, producing meaningful error messages.

The try/except Structure

Here's our problematic code from before:

num1 = 10
num2 = 0
result = num1 / num2
print("Result:", result)

Now let's add exception handling:

try:
    num1 = 10
    num2 = 0
    result = num1 / num2
    print("Result:", result)
except ZeroDivisionError:
    print("Error: Cannot divide by zero!")

When we run this improved code:

Error: Cannot divide by zero!

✅ The program doesn't crash!
✅ We get a friendly error message
✅ The program can continue running

We can handle multiple exception types:

try:
    # Code that might cause exceptions
    value = int(input("Enter a number: "))
    result = 10 / value
except ValueError:
    print("Please enter a valid number!")
except ZeroDivisionError:
    print("Cannot divide by zero!")

✨ Benefits of Exception Handling

Why Use Exception Handling?

❌ Without Exception Handling
  • Program crashes unexpectedly
  • Cryptic error messages
  • Lost data and progress
  • Poor user experience
  • Difficult to debug
✅ With Exception Handling
  • Program continues running
  • Meaningful error messages
  • Data can be saved
  • Better user experience
  • Easier maintenance

⚡ Best Practices & Common Mistakes

Common Mistakes to Avoid

❌ Don't Do This:
try:
    # Hundreds of lines of code here
    # Makes it hard to find the error
except:
    pass # Silently ignores all errors!
✅ Do This Instead:
try:
    # Only the risky operation
    result = risky_operation()
except SpecificError as e:
    print(f"Error occurred: {e}")
    # Handle the error appropriately
