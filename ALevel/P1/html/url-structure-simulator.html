Protocol - How to connect

What is a Protocol?

The protocol tells your browser HOW to communicate with the server. Think of it like choosing between sending a letter (http) or a secure registered mail (https).

http://
https://
ftp://
file://

Subdomain - Section of website (optional)

What is a Subdomain?

Subdomains are like different departments in a company. 'www' is the most common, but you might also see 'blog', 'shop', or 'mail'.

None
www.
blog.
shop.
mail.

Domain - The website's name

What is a Domain?

The domain is the website's unique name on the internet. It's like a street address that helps people find your website.

example.com
school.edu
company.co.uk
mysite.org
shop.net

Port - The door number (optional)

What is a Port?

Ports are like door numbers on a building. Web traffic usually uses port 80 (http) or 443 (https), but these are hidden. Custom ports must be specified.

Default
:8080
:3000
:8888

Path - Location of the resource

What is a Path?

The path shows where to find a specific page or file on the website. It's like navigating through folders on your computer.

None
/ (homepage)
about
products/laptops
blog/2024/article
index.html

Query - Extra information (optional)

What are Query Parameters?

Query parameters send extra information to the website, like search terms or filters. They start with ? and use & between multiple parameters.

None
?search=laptop
?id=123
?page=2&sort=price

Fragment - Jump to section (optional)

What is a Fragment?

Fragments (or anchors) let you jump to a specific part of a page. They start with # and don't get sent to the server.

None
top
section2
comments
