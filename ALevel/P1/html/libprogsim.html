Library Programs: An Interactive Simulation

What is a Library Routine?

Imagine you're building a complex project. Instead of making every single component from scratch, you use pre-made, expertly crafted parts. In programming, these parts are called library routines.

They are pre-written, pre-tested, and compiled blocks of code that perform common or complex tasks. You can simply 'link' or 'import' them into your program.

This simulation will guide you through why, and when, we use them. Click 'Next' to begin.

The Programmer's Task

A programmer needs to write a program that sorts a list of 10,000 numbers. This is a common but complex task. There are two ways to approach this...

// Task: Sort a large array of numbers
let hugeArray = [934, 102, 5, ...]; // 10,000 numbers

Approach 1: Code from Scratch

The programmer could write their own sorting algorithm. This requires deep knowledge, a lot of code, and extensive testing.

function bubbleSort(arr) {
  let n = arr.length;
  for (let i = 0; i < n - 1; i++) {
    for (let j = 0; j < n - i - 1; j++) {
      if (arr[j] > arr[j + 1]) {
        let temp = arr[j];
        arr[j] = arr[j + 1];
        arr[j + 1] = temp;
      }
    }
  }
  return arr;
}
// And this is just ONE simple algorithm...

Approach 2: Use a Library Routine

Or, the programmer can import a pre-written, highly optimized sorting function from a library. The complex code is replaced by a single, simple function call.

→
// Import routine from a trusted library
import { sort } from 'optimised-routines';

// Call the routine
let sortedArray = sort(hugeArray);
