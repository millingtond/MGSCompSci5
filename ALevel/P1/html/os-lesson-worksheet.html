Interrupt Handling Process:

  1. Check for interrupt flag during fetch-execute cycle
  2. If interrupt detected, halt current process
  3. Save current process values on stack
  4. Load and execute appropriate ISR (Interrupt Service Routine)
  5. Handle multiple interrupts using priorities
  6. Restore original process and continue

📝 Summary & Final Quiz

Key Takeaways

  • Essential Role: Hardware is unusable without an OS
  • Interface: OS acts as intermediary between user and hardware
  • Management: Handles memory, files, devices, processes, and security
  • Interrupts: Manages hardware/software interrupts with priorities
  • Multi-tasking: Enables multiple processes to run efficiently

📋 Exam Practice Questions

Question 1 (6 marks)

Explain three ways an operating system manages memory, including how virtual memory works.

Mark Scheme:
  • Allocates/deallocates RAM to processes (1 mark)
  • Keeps track of free/allocated memory (1 mark)
  • Virtual memory allows programs larger than RAM to run (1 mark)
  • Swaps data between RAM and hard drive (1 mark)
  • Uses paging/segmentation techniques (1 mark)
  • Provides memory protection between processes (1 mark)

Question 2 (4 marks)

Describe the steps involved in interrupt handling during the fetch-execute cycle.

Mark Scheme:
  • Check for interrupt flag/bit in interrupt register (1 mark)
  • Halt execution of current process (1 mark)
  • Store current process values on stack/register (1 mark)
  • Load and execute appropriate ISR code (1 mark)

Question 3 (5 marks)

Compare GUI and CLI interfaces, giving two advantages of each.

Mark Scheme:

1 mark for comparison, 2 marks for GUI advantages, 2 marks for CLI advantages
