üîß TLS Fundamentals

What is TLS?

TLS (Transport Layer Security) provides secure communication over a network by:

  • Maintaining data integrity
  • Providing an additional layer of security
  • Protecting information using encryption
  • Allowing authentication of servers and clients

Handshake Steps:

1. Client requests server identification
2. Server sends digital certificate with public key
3. Client verifies certificate against trusted CAs
4. Client sends its certificate (if requested)
5. Client sends encrypted message using server's public key
6. Server decrypts message with private key
7. Both create symmetric session key
8. Exchange signed acknowledgments

üîê Encryption Methods in TLS

TLS uses both asymmetric and symmetric encryption for different purposes.

Compare Encryption Types

Asymmetric Encryption

Keys: Public and Private pair

Speed: Slower

Use in TLS: Initial handshake

Purpose: Key exchange & authentication

Symmetric Encryption

Keys: Single shared key

Speed: Faster

Use in TLS: Data transmission

Purpose: Bulk data encryption

üìú Digital Certificates

Digital certificates are crucial for authentication in TLS. They contain the server's public key and are issued by trusted Certificate Authorities (CAs).

Sample Digital Certificate

Subject: www.example-bank.com
Issuer (CA): TrustedCA Corporation
Valid From: 01/01/2024
Valid To: 01/01/2025
Public Key: [RSA 2048-bit key]
Signature Algorithm: SHA256 with RSA
