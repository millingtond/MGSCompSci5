üîß Understanding RPN Basics

What is Reverse Polish Notation?

Reverse Polish Notation (RPN), also known as postfix notation, is a mathematical notation where operators follow their operands. This eliminates the need for brackets and operator precedence rules.

Feature Infix Notation RPN (Postfix)
Operator Position Between operands (3 + 4) After operands (3 4 +)
Brackets Required? Yes, for complex expressions Never needed
Precedence Rules? Yes (BIDMAS/PEMDAS) Not needed
Ambiguity Can be ambiguous without brackets Always unambiguous
Evaluation Complex parsing required Simple left-to-right scan

RPN Evaluation Algorithm

1. Scan the expression from left to right
2. If the symbol is a number, push it onto the stack
3. If the symbol is an operator:
a. Pop the required operands (2 for binary operators)
b. Apply the operator (note: order matters for - and √∑)
c. Push the result back onto the stack
4. The final result is the only item left on the stack

üöÄ Real-World Applications

Where is RPN Used?

1. Calculator Design

Many scientific calculators (especially HP calculators) use RPN because:

  • No need for parentheses keys
  • Fewer keystrokes for complex calculations
  • See intermediate results
  • More efficient memory usage

2. Compiler Design

Compilers convert infix expressions to RPN because:

  • Easier to generate machine code
  • Natural fit for stack-based architectures
  • No ambiguity in evaluation order
  • Simplified expression parsing

Question 2 (6 marks)

Convert the following infix expression to RPN and show the stack contents at each step of evaluation:

5 √ó (3 + 2) - 4

Mark Scheme:

Conversion (2 marks):

RPN: 5 3 2 + √ó 4 -

Evaluation steps (4 marks - 0.5 per step):

  1. Read 5: Stack = [5]
  2. Read 3: Stack = [5, 3]
  3. Read 2: Stack = [5, 3, 2]
  4. Read +: Pop 2,3, push 5: Stack = [5, 5]
  5. Read √ó: Pop 5,5, push 25: Stack = [25]
  6. Read 4: Stack = [25, 4]
  7. Read -: Pop 4,25, push 21: Stack = [21]
  8. Result = 21
