🔑 Public Key Encryption

Public key encryption uses two different keys - one public and one private. Let's see how it works!

Step-by-Step Simulation

Step 1: Key Generation

Alice generates a pair of keys:

Public
Private

The public key can be shared with anyone. The private key must be kept secret!

Step 2: Bob Gets Alice's Public Key

Bob wants to send Alice a secret message. He obtains her public key.

Alice's Public Key: Public

This key is freely available - anyone can have it!

Step 3: Bob Encrypts the Message

Bob uses Alice's public key to encrypt his message:

Plain text: "Meet at 3pm"
+ Alice's Public Key
= Cipher text: "X#k9@m$2!p"
Step 4: Alice Decrypts the Message

Only Alice's private key can decrypt the message:

Cipher text: "X#k9@m$2!p"
+ Alice's Private Key
= Plain text: "Meet at 3pm"

Even if someone intercepts the cipher text, they can't read it without the private key!

📝 Summary & Final Quiz

Key Takeaways

  • Encryption transforms plain text into cipher text using an algorithm
  • Plain text is the original, readable message
  • Cipher text is the encrypted, unreadable version
  • Cryptography ensures: confidentiality, authentication, integrity, and non-repudiation
  • Public key encryption uses two different keys - public and private
  • Private keys must be kept secret and are used for decryption
  • Asymmetric encryption uses different keys; symmetric uses the same key
