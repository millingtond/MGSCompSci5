What is Dijkstra's Algorithm?

Dijkstra's algorithm is a graph search algorithm that solves the single-source shortest path problem for a graph with non-negative edge weights. It was conceived by computer scientist Edsger W. Dijkstra in 1956.

Key Applications:

  • GPS navigation systems
  • Network routing protocols
  • Flight path planning
  • Social network analysis

How Dijkstra's Algorithm Works

The algorithm maintains a set of vertices whose shortest distance from the source is known. It repeatedly selects the vertex with the minimum distance and updates distances to its neighbors.

Algorithm Steps:

1. Initialize distances: Set source to 0, all others to infinity
2. Create a set of unvisited vertices
3. Select the unvisited vertex with minimum distance
4. Update distances to its unvisited neighbors
5. Mark current vertex as visited
6. Repeat until all vertices are visited or target is reached

// Dijkstra's Algorithm Pseudocode
FUNCTION Dijkstra(Graph, source):
  // Initialize distances
  FOR EACH vertex v IN Graph:
    distance[v] ‚Üê INFINITY
    previous[v] ‚Üê UNDEFINED
  
  distance[source] ‚Üê 0
  unvisited ‚Üê SET OF ALL vertices
  
  WHILE unvisited IS NOT EMPTY:
    current ‚Üê vertex IN unvisited WITH minimum distance[current]
    REMOVE current FROM unvisited
    
    FOR EACH neighbor OF current:
      alt ‚Üê distance[current] + weight(current, neighbor)
      IF alt < distance[neighbor]:
        distance[neighbor] ‚Üê alt
        previous[neighbor] ‚Üê current
  
  RETURN distance, previous

üí° Hint: Think about what "non-negative" means - it includes zero but excludes negative numbers. The algorithm starts with infinite distances because we haven't found any paths yet!

GPS Navigation Systems

Modern GPS systems use variations of Dijkstra's algorithm to find the shortest route between locations. The graph represents the road network where:

  • Vertices are intersections or points of interest
  • Edges are road segments
  • Weights can represent distance, time, or fuel consumption

Mistake #1: Forgetting to Update All Neighbors

When visiting a vertex, you must check ALL unvisited neighbors, not just the first one you find.

‚ùå Wrong: Update only the closest neighbor
‚úì Correct: Check all neighbors and update if a shorter path is found

Other Common Mistakes

  • Using with negative weights: Dijkstra's doesn't work correctly with negative edge weights
  • Not initializing properly: Forgetting to set source distance to 0 or others to infinity
  • Revisiting vertices: Once marked as visited, a vertex should not be reconsidered
  • Wrong comparison: Using ‚â§ instead of < when comparing distances

üìù Summary & Final Quiz

Key Takeaways

  • Purpose: Finds shortest paths from a source to all other vertices
  • Requirements: Non-negative edge weights
  • Time Complexity: O((V + E) log V) with binary heap
  • Greedy Algorithm: Always selects minimum distance vertex
  • Applications: GPS, networking, gaming, and more
