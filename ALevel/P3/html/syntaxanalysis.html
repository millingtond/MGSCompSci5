What is Syntax Analysis?

After Lexical Analysis breaks code into tokens, Syntax Analysis checks if the sequence of these tokens follows the grammatical rules of the programming language.

Think of it like checking if an English sentence is grammatically correct. The "words" (tokens) must be in a valid order to make sense.

Key Functions:

  • Checks tokens against the language's grammar rules.
  • Reports any syntax errors it finds.
  • Produces a Parse Tree (or Abstract Syntax Tree) if the code is valid.

The Parser in Action

The Parser takes the stream of tokens from the Lexical Analyser and checks them against the language's rules.

# Simple line of Python code

score = 10

# Token Stream from Lexical Analysis

How is a Parse Tree Structured?

Let's break down the structure for score = 10. Think of it like a family tree for code.

1

The Root (The Main Action)

The most important operation is the assignment (`=`). It's the main action, so it goes at the top.

2

The Branches (The Details)

The assignment needs a variable (`score`) and a value (`10`). These become the "children," branching off from the root.

3

The Leaves (The End Points)

`score` and `10` are the final components. They can't be broken down further, so they are the leaves at the bottom.

Handling Syntax Errors

What happens if the code violates the language's grammar? The parser stops and reports a syntax error.

# Invalid Python code

score 10 =
SyntaxError: invalid syntax
