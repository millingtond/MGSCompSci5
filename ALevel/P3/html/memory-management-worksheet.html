📄 Paging

Definition: Paging is reading/writing same-size blocks of data from/to secondary storage when required.

How Paging Works

Memory is divided into fixed-size blocks called pages. The operating system manages these pages using a page table.

🧩 Segmentation

Definition: Logical address space is broken up into variable-size memory blocks/partitions called segments.

Key Concepts

  • Each segment has a name and size
  • Segments from logical memory are loaded into physical memory for execution
  • Address contains segment name/number and offset value
  • Segment map table maps logical/virtual addresses to physical addresses

📝 Summary & Final Quiz

Key Takeaways

  • Multitasking: Managing execution of multiple programs through time-sharing
  • Paging: Fixed-size memory blocks with simple management
  • Segmentation: Variable-size blocks matching logical program structure
  • Trade-offs: Paging avoids external fragmentation, segmentation provides logical separation

Question 1 (6 marks)

Explain the difference between paging and segmentation, including:

  • How memory is divided (2 marks)
  • Address translation process (2 marks)
  • One advantage of each method (2 marks)
Mark Scheme:

Memory Division (2 marks):

  • Paging: Fixed-size blocks called pages (1 mark)
  • Segmentation: Variable-size blocks called segments (1 mark)

Address Translation (2 marks):

  • Paging: Page number + offset, uses page table (1 mark)
  • Segmentation: Segment number + offset, uses segment table (1 mark)

Advantages (2 marks):

  • Paging: No external fragmentation (1 mark)
  • Segmentation: Logical program structure / protection (1 mark)

Question 2 (4 marks)

A computer system uses paging with a page size of 2KB. Calculate the page number and offset for the logical address 6500. Show your working.

Mark Scheme:
  • Convert page size to bytes: 2KB = 2048 bytes (1 mark)
  • Calculate page number: 6500 ÷ 2048 = 3 (1 mark)
  • Calculate offset: 6500 % 2048 = 356 (1 mark)
  • Clear final answer: Page 3, Offset 356 (1 mark)
