📐 Understanding Heuristics

A heuristic is an estimation of the cost from a node to the goal. Good heuristics make A* efficient!

Common Heuristic Functions

Heuristic Formula Best Use Case
Manhattan Distance |x₁ - x₂| + |y₁ - y₂| Grid with 4-way movement
Euclidean Distance √((x₁ - x₂)² + (y₁ - y₂)²) Any direction movement
Diagonal Distance max(|x₁ - x₂|, |y₁ - y₂|) 8-way grid movement

⚖️ A* vs Dijkstra's Algorithm

A* Algorithm

  • Uses heuristic function h(n)
  • Explores fewer nodes
  • Faster in practice
  • Requires goal knowledge
  • Best for single target
  • f(n) = g(n) + h(n)

Dijkstra's Algorithm

  • No heuristic used
  • Explores all directions
  • Slower but thorough
  • No goal knowledge needed
  • Finds all shortest paths
  • Priority = g(n) only

💻 A* Implementation

Pseudocode

// A* Pathfinding Algorithm
FUNCTION AStar(start, goal):
  openList = [start]
  closedList = []
  start.g = 0
  start.h = HEURISTIC(start, goal)
  start.f = start.g + start.h

  WHILE openList is not empty:
    current = node with lowest f in openList
    IF current == goal:
      RETURN reconstructPath(current)

    openList.REMOVE(current)
    closedList.ADD(current)

    FOR EACH neighbor of current:
      IF neighbor in closedList:
        CONTINUE

      tentative_g = current.g + distance(current, neighbor)

      IF neighbor not in openList:
        openList.ADD(neighbor)
      ELSE IF tentative_g >= neighbor.g:
        CONTINUE

      neighbor.parent = current
      neighbor.g = tentative_g
      neighbor.h = HEURISTIC(neighbor, goal)
      neighbor.f = neighbor.g + neighbor.h

  RETURN failure

Common Mistakes

  • Using an inadmissible heuristic (overestimating)
  • Not updating g values when finding better paths
  • Forgetting to check the closed list
  • Using wrong distance calculation for grid type

⚡ Optimization & Advanced Concepts

Time Complexity

Algorithm Time Complexity Space Complexity
A* (with good heuristic) O(b^d) O(b^d)
Dijkstra O(V²) O(V)

Where b = branching factor, d = depth of solution, V = vertices

📝 Summary & Final Quiz

Key Takeaways

  • A* combines actual cost g(n) with heuristic h(n)
  • f(n) = g(n) + h(n) determines node priority
  • Heuristics guide search toward the goal
  • Admissible heuristics never overestimate
  • A* is faster than Dijkstra for single targets
  • Trade-off: Speed vs completeness
