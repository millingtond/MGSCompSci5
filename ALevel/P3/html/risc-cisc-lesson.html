üèõÔ∏è CISC Architecture

Complex Instruction Set Computer

CISC processors are designed to complete tasks with as few lines of assembly code as possible, using complex instructions that can perform multiple operations.

CISC Characteristics

  • Very large instruction set with complex operations
  • Variable length instruction formats
  • Fewer general purpose registers (more RAM usage)
  • Micro-programmed with ROM storing microcode
  • Instructions may take multiple clock cycles

‚ö° RISC Architecture

Reduced Instruction Set Computer

RISC processors use a small set of simple instructions, each designed to execute in a single clock cycle, enabling efficient pipelining.

RISC Characteristics

  • Small instruction set of simple operations
  • Fixed length instruction formats
  • More general purpose registers
  • Hard-wired control unit (no microcode)
  • Single cycle execution (when possible)

üîÑ Introduction to Pipelining

What is Pipelining?

Pipelining is a technique where multiple instructions are overlapped in execution, similar to an assembly line in a factory. It's a form of instruction-level parallelism.

Pipelining Analogy: Laundry

Imagine doing laundry with these stages: Wash ‚Üí Dry ‚Üí Fold ‚Üí Store

Without Pipelining: Complete all stages for Load 1 before starting Load 2

Time: 4 stages √ó 4 loads = 16 time units

With Pipelining: Start Load 2's wash while Load 1 is drying

Time: 4 stages + 3 additional loads = 7 time units

Data Dependencies

Pipeline hazards occur when instructions cannot execute in the expected order, causing the pipeline to stall.

Example: Data Dependency

Consider these instructions:

ADD R1, R2, R3 ; R1 = R2 + R3
SUB R4, R1, R5 ; R4 = R1 - R5 (needs R1 from previous instruction)

Problem: The SUB instruction needs the result from ADD, but ADD hasn't written back yet!

What is a PCB?

A Process Control Block is a data structure in memory that stores all information about an active process. The OS creates a PCB when a process starts.

Key Takeaways

  • CISC uses complex instructions, fewer registers, and microcode
  • RISC uses simple instructions, more registers, and hard-wired control
  • Pipelining enables instruction-level parallelism
  • Pipeline hazards can stall execution but have solutions
  • Interrupts temporarily halt execution for higher priority tasks
  • PCBs store process information for context switching
  • Interrupt handling is complex with pipelining
