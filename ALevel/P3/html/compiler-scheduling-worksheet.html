Lexical Analysis

Lexical analysis is the first stage of compilation. It prepares the code for reading by:

  • Removing spaces and comments
  • Checking for illegal operators and data types
  • Creating tokens from the symbol table
  • Converting code into a tokenized format

Step 2: Identify keywords (int)

Step 4: Identify operators (=, +)
Step 5: Create token stream

Syntax Analysis

Syntax analysis (parsing) is the second stage. It:

  • Checks that code matches the language grammar
  • Ensures tokens conform to programming language rules
  • Reports syntax errors
  • Produces a parse tree

Low-Level Scheduler

The low-level scheduler manages CPU time allocation to processes. It:

  • Directly interacts with the CPU and kernel
  • Ensures fair CPU time distribution
  • Maintains system responsiveness
  • Uses algorithms like round-robin or priority-based scheduling
  • Handles context switching between processes

Key Takeaways

  • Optimization: Improves code efficiency

  • Low-Level Scheduler: Manages CPU time and process switching
