üìä Types of Test Data

To thoroughly test a program, we need different types of test data:

Normal Data

Typical, valid data that the program should accept

Example: Age = 25 (for age 0-120)

Boundary Data

Data at the edges of valid ranges

Example: Age = 0, 120 (min and max valid ages)

Invalid Data

Data of correct type but outside valid range

Example: Age = -5 or 150

Erroneous Data

Data of wrong type

Example: Age = "twenty" (text instead of number)

‚ùå Mistake #1

Testing Only Happy Path

‚úì Solution

Problem: Only testing when everything goes right

Fix: Test error conditions, edge cases, and invalid inputs

Example: Don't just test login with correct password - test wrong password, empty fields, SQL injection attempts

‚ùå Mistake #2

No Boundary Testing

‚úì Solution

Problem: Missing tests at limits of valid ranges

Fix: Always test minimum, maximum, and just outside values

Example: For age 0-120, test: -1, 0, 1, 119, 120, 121

‚úì Solution

Problem: Waiting until everything is done to test

Fix: Use iterative testing - test each module as you build it

Benefit: Bugs are cheaper and easier to fix when found early

üìù Practice Exam Questions

Question 1 (4 marks)

Define and give an example of each type of test data:

a) Normal data (1 mark)

b) Boundary data (1 mark)

c) Invalid data (1 mark)

d) Erroneous data (1 mark)

Mark Scheme:

  • Normal data: Typical valid data that the program should accept. Example: Age = 25 for a range of 0-120 (1 mark)
  • Boundary data: Data at the edges of valid ranges. Example: Age = 0 and 120 for a range of 0-120 (1 mark)
  • Invalid data: Data of correct type but outside valid range. Example: Age = -5 or 150 for a range of 0-120 (1 mark)
  • Erroneous data: Data of wrong type. Example: Age = "twenty" when expecting a number (1 mark)

Question 2 (3 marks)

Explain the difference between iterative testing and final/terminal testing. Give one advantage of iterative testing over final testing.

Mark Scheme:

  • Iterative testing: Testing done repeatedly throughout development as each module is completed (1 mark)
  • Final/Terminal testing: Testing done at the end when all modules are complete (1 mark)
  • Advantage: Bugs are found early and are cheaper/easier to fix OR errors can be corrected before they affect other modules (1 mark)

Question 3 (2 marks)

A syntax error prevents a program from running. A logic error allows the program to run but produces incorrect results. Give one example of each type of error.

Mark Scheme:

  • Syntax error example: Missing colon after if statement / Unclosed bracket / Incorrect indentation / Misspelled keyword (1 mark for any valid example)
  • Logic error example: Using + instead of * / Wrong formula / Off-by-one error in loop / Wrong comparison operator (1 mark for any valid example)

Question 4 (3 marks)

A program validates user passwords. The password must be between 8 and 20 characters long. Create a test plan with three different test cases, each using a different type of test data.

Mark Scheme:

Award 1 mark for each valid test case (max 3) that includes test data and identifies the type:

  • Normal: Password = "MyPass123" (9 characters) - Should accept
  • Boundary: Password = "Pass1234" (8 characters) or "12345678901234567890" (20 characters) - Should accept
  • Invalid: Password = "Short1" (6 characters) or "ThisPasswordIsTooLong123" (24 characters) - Should reject
  • Erroneous: Password = 12345678 (number instead of string) - Should give error

Question 5 (2 marks)

State what is meant by a test plan and explain one benefit of using a test plan when developing software.

Mark Scheme:

  • Definition: A structured document that includes test data and expected results for testing a program (1 mark)
  • Benefit: Helps find logic errors / Prevents runtime errors / Ensures all conditions are tested / Makes testing systematic and repeatable (1 mark for any valid benefit)

üîë Key Points to Remember
