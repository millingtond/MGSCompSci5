Simplified SQL Injection Example:

Imagine a website login uses this SQL query (simplified):

SELECT * FROM users WHERE username = '[user_input_username]' AND password = '[user_input_password]';

If an attacker enters the following into the username field (and anything in password):

' OR '1'='1

The query might become:

SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '...';

Because '1'='1' is always true, this could bypass the password check and log the attacker in as the first user in the database (often an admin!).

Task: Identify the Attack!

1. You receive an urgent email from "your bank" asking you to click a link and verify your account details due to suspicious activity. The website looks identical to your bank's site.

2. A popular online gaming service suddenly becomes extremely slow and then completely inaccessible for several hours due to an overwhelming amount of traffic from thousands of computers.

3. An attacker modifies a website's search query in the URL to include database commands, tricking the server into revealing all user email addresses.

4. You type your bank's correct web address, but the site you land on looks slightly different and your browser shows a security warning. You later find unauthorized transactions.

5. While using a public Wi-Fi network at a coffee shop, you log into your email. Later, you discover someone has accessed your emails without your permission.

Exam Practice Questions

Apply your knowledge. Mark schemes unlock after you type enough!

1. Explain what Phishing is and give one example of how an attacker might carry it out. [3 marks]

Mark Scheme:

  • Phishing is a form of social engineering (1 mark).
  • Where attackers attempt to acquire sensitive information (e.g., usernames, passwords, credit card details) by masquerading as a trustworthy entity (1 mark).
  • Example: Sending a fraudulent email that appears to be from a bank, asking the user to click a link and enter their login details on a fake website (1 mark). (Allow other valid examples)

2. Describe the main goal of a Denial of Service (DoS) attack. [2 marks]

Mark Scheme:

  • To make a machine or network resource unavailable to its intended users (1 mark).
  • Achieved by overwhelming the target system with a flood of internet traffic or requests (1 mark).

3. Explain how an SQL Injection attack works and state one potential consequence. [3 marks]

Mark Scheme:

  • An attacker inserts malicious SQL code into an input field of a web application (1 mark).
  • This code is then executed by the database server (1 mark).
  • Potential consequence: Unauthorized access to/theft of data OR modification/deletion of data OR gaining administrative control of the database (1 mark).

Key Takeaways

  • Social Engineering relies on manipulating people to gain access or information.
  • Phishing uses deceptive communications (emails, messages) to steal sensitive data.
  • Pharming redirects users to fake websites by compromising DNS or host files.
  • Denial of Service (DoS) attacks aim to make services unavailable by overwhelming them with traffic.
  • Data Interception involves capturing data as it's transmitted over a network, often on unsecure connections.
  • SQL Injection exploits vulnerabilities in web applications to execute malicious database commands.
