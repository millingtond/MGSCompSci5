Introduction to Primary Storage

What is Primary Storage?

Primary storage, also known as main memory, is the computer's working space. Let's explore its key characteristics:

Primary storage is:

  • ✅ Directly accessible by the CPU
  • ✅ Volatile (loses data when power is off)
  • ✅ Fast to access and retrieve data
  • ✅ Limited in size compared to secondary storage

ROM: Read Only Memory

Understanding ROM

ROM is like a locked filing cabinet - it contains important information that doesn't change.

Key Facts about ROM:

  • 📌 Non-volatile - Keeps data even when power is off
  • 📌 Read-only - Cannot be changed during normal operation
  • 📌 Stores BIOS - Basic Input Output System
  • 📌 Boot instructions - Tells computer how to start up

Real World Applications

Storage in Action

Let's explore how different types of storage are used in real devices and situations.

🎮 Gaming Console

How does a PlayStation or Xbox use different storage types?

  • RAM: Loads game assets for smooth gameplay
  • ROM: Stores system firmware
  • SSD/HDD: Stores games and save files

📱 Smartphone

Your phone uses storage cleverly:

  • RAM: Keeps apps running in background
  • ROM: Stores bootloader and recovery
  • Flash Storage: Stores apps, photos, videos

🚗 Modern Car

Even cars use computer storage:

  • RAM: Navigation and infotainment
  • ROM: Engine control firmware
  • Storage: Maps and media files

Common Mistakes to Avoid

⚠️ Watch Out For These!

Mistake 1: Confusing RAM with Storage

Wrong: "I need more RAM to store my photos"

Right: "I need more storage (HDD/SSD) for my photos. RAM is for running programs."

Mistake 2: Thinking ROM can be easily changed

Wrong: "I'll just update the ROM like I update apps"

Right: "ROM typically can't be changed. It stores permanent instructions like BIOS."

Mistake 3: Not understanding volatile vs non-volatile

Wrong: "My work is safe in RAM even if power goes off"

Right: "RAM is volatile - always save important work to storage!"

Mistake 4: Virtual memory as a RAM replacement

Wrong: "Virtual memory is just as good as RAM"

Right: "Virtual memory is much slower than RAM and should be a last resort."

Practice Exam Questions

Question 1 (2 marks)

State two differences between RAM and ROM.

Mark Scheme:

1 mark for each valid difference (max 2):

  • RAM is volatile, ROM is non-volatile
  • RAM can be written to, ROM is read-only
  • RAM stores running programs/data, ROM stores boot instructions/BIOS
  • RAM contents change frequently, ROM contents rarely/never change
  • RAM is easily expandable, ROM size is fixed

Question 2 (3 marks)

Explain how virtual memory works when RAM is full.

Mark Scheme:

1 mark for each point (max 3):

  • Part of secondary storage/hard drive is used as virtual memory
  • Data/programs are moved (swapped) from RAM to virtual memory
  • This frees up space in RAM for new programs/processes
  • When needed, data is swapped back from virtual memory to RAM
  • Virtual memory is slower than RAM as it uses secondary storage

Question 3 (4 marks)

A computer has 4GB of RAM. Describe two ways to improve its performance and explain why each would help.

Mark Scheme:

2 marks for each improvement (1 for stating, 1 for explanation):

  • Add more RAM: Allows more programs to run simultaneously, reduces use of slower virtual memory
  • Upgrade to SSD: Faster read/write speeds than HDD, programs load quicker
  • Increase CPU clock speed: More instructions processed per second, faster execution
  • Add more CPU cores: Better multitasking, can run multiple processes in parallel
  • Increase cache size: More frequently used data stored close to CPU for faster access

Question 4 (6 marks)

A video editing company is experiencing performance issues. Their computers have 8GB RAM but employees work with large 4K video files. The IT manager suggests either adding more RAM or switching from HDD to SSD storage.

Evaluate both options and recommend which upgrade would be most beneficial. Justify your answer.

Mark Scheme:

Points for consideration (max 6 marks):

  • RAM upgrade benefits: Large video files need lots of RAM for processing, reduces virtual memory use, smoother playback/editing
  • SSD upgrade benefits: Faster loading of video files, quicker saves, better scrubbing through timeline
  • Evaluation: Both would help, but RAM likely more critical for active editing
  • Recommendation: Clear choice with justification
  • Consider both: Ideal solution might be both upgrades if budget allows
  • Alternative considerations: GPU upgrade for rendering, specific amounts (e.g., 32GB RAM)
