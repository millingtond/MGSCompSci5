Wide Area Networks (WAN)

What is a WAN?

A Wide Area Network (WAN) is a network that spans a large geographical area. The communication medium (cables that connect networks together) is owned by telecommunications companies like BT.

Local Area Networks (LAN)

Understanding LANs

A Local Area Network (LAN) is where computers are connected to each other and use MAC addresses to communicate. LANs are restricted to a small geographical area/site and the hardware (routers, WAPs, switches) is owned by the company/home owners.

Client-Server Networks

What is a Client-Server Network?

A network that uses central computers, known as servers, to enable resource sharing and to facilitate communication between the other computers on the network.

Peer-to-Peer Networks

Understanding Peer-to-Peer Networks

In a peer-to-peer network:

  • No server required
  • Computers are directly connected to each other
  • Computers are independent
  • Software installed/updated individually
  • No central security or file storage

Common Mistakes to Avoid

Spot the Mistakes

Can you identify what's wrong with these statements?

Mistake Identification

Statement 1: "LANs can connect computers in different countries."

Statement 2: "In a peer-to-peer network, all files are stored on a central server."

Statement 3: "WANs use infrastructure owned by the company using the network."

Statement 4: "Client-server networks make it easier to manage security centrally."

Misconception Buster

‚ö†Ô∏è Common Misconception

Myth: "Peer-to-peer networks are always better because they're cheaper."

Reality: While P2P networks have lower initial costs, they lack central management, security, and backup features. For businesses with important data or many users, client-server networks are often more suitable despite higher costs.

Key Takeaways

üéØ Essential Points to Remember

WAN (Wide Area Network)

  • Spans large geographical areas
  • Uses telecom company infrastructure
  • Connects different sites/cities/countries

LAN (Local Area Network)

  • Limited to small area/single site
  • Uses MAC addresses
  • Hardware owned by users

Client-Server Networks

  • Central server manages resources
  • Better security and backups
  • Higher cost but easier management

Peer-to-Peer Networks

  • No central server
  • Lower cost, simpler setup
  • Good for small networks
